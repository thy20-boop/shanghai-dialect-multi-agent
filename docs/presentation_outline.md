# 汇报提纲

## 1. 问题

通用 ASR 容易把上海话普通话化、漏掉方言词，并在长视频、短口语、地名人名和混说术语时产生错误。单一模型即使很强，也缺少可解释的质量控制和持续改进机制。

## 2. 数据

使用公开上海话会话语音数据集，固定 3700 条训练、92 条验证，并比较本地 LoRA、Dolphin 方言 ASR 与多智能体协同输出。

## 3. 方法

```text
音频/视频输入
-> 音频预处理智能体
-> Dolphin ASR 专家主识别
-> 本地 Whisper-LoRA 复核/兜底
-> 候选仲裁智能体
-> 纠错记忆智能体
-> 上海话转普通话智能体
-> 风险检测智能体
-> 主动学习智能体
```

## 4. 展示

- 输入上海话音频。
- 展示原始转写、普通话输出与自动修复。
- 展示 `阿拉`、`侬`、`搿个`、`勿` 等线索。
- 展示 Dolphin 与 LoRA 候选分歧、智能体协作记录、待复核片段和主动学习样本。
- 展示质量评分、候选共识评分，以及主动学习队列如何进入下一轮训练。
- 展示 MP3 语音输出：老人不识字时，可以直接听普通话结果或吴语口语稿。
- 展示 Codex 联网回答：用户用上海话提问，系统识别问题并生成 Codex 任务包；Codex 搜索网页、给出带来源回答，并生成普通话 MP3 与吴语口语稿 MP3。

## 5. 指标

- CER
- 方言词召回率
- 术语召回率
- 高风险片段拦截
- 候选分歧发现数量
- 主动学习样本沉淀数量
- 质量评分分布
- 人工确认后可导出的再训练样本数

## 6. 局限

上海话书写规范不统一，规则词典覆盖有限，Dolphin 和 LoRA 都会在特定口音、长视频和背景声下出错。系统用风险检测和主动学习降低误用风险，但真实部署仍需要母语者评估。
